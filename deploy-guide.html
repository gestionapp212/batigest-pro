🔧 Créer et déployer le Worker Cloudflare
Installation de Wrangler CLI + déploiement du fichier workers/gestapp-api/src/index.js
En cours
Vous avez besoin de Node.js (v18+) et d'un compte Cloudflare gratuit. Ces commandes sont à exécuter dans votre terminal, dans le dossier racine du projet.
📦 Installer Wrangler CLI
npm install -g wrangler
🔑 Se connecter à Cloudflare
wrangler login
Un navigateur s'ouvre pour autoriser Wrangler. Cliquez "Allow" dans Cloudflare Dashboard.
🚀 Déployer le Worker (depuis la racine du projet)
cd workers/gestapp-api wrangler deploy
Erreur normale à cette étape : Le Worker va se déployer mais les requêtes à la DB échoueront tant que D1 n'est pas configuré. C'est normal !
Format attendu : https://gestapp-api.[sous-domaine].workers.dev
2
🗄️ Créer la base de données D1 (SQLite)
Initialise toutes les tables : platform_users, companies, clients, devis, factures…
En attente
Cloudflare D1 est une base SQLite serverless gratuite jusqu'à 5 Go. Elle remplace entièrement Supabase PostgreSQL.
📦 Créer la base D1
wrangler d1 create gestapp-db
Le terminal affiche un bloc comme :
[[d1_databases]]
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
Copiez ce database_id ci-dessous.
📋 Initialiser le schéma (toutes les tables)
wrangler d1 execute gestapp-db --file=schema.sql --remote
Cette commande crée automatiquement : platform_users, companies, clients, devis, factures, paiements, chantiers, stock, fournisseurs, garanties, agenda, taches, pipeline, galerie, audit_logs
3
👑 Créer le compte SuperAdmin
Génère le hash PBKDF2 du mot de passe et insère le compte dans D1
En attente
Renseignez l'email et le mot de passe du SuperAdmin. Le SQL sera généré automatiquement.
⚠️ Changez le mot de passe après la première connexion !
⚙️ Générer le hash et le SQL d'insertion
node seed.js "VotreMDP2025!"
Le terminal affiche un SQL INSERT. Copiez-le entier et collez-le dans le champ ci-dessous.
Ou utilisez le générateur intégré ci-dessous pour obtenir le SQL directement.
Cette génération se fait côté navigateur via Web Crypto API (identique au Worker) 

        

          
          
        

      


      
▶️ Exécuter le SQL dans D1 (via CLI)

      

        
# Coller le SQL généré ici :
wrangler d1 execute gestapp-db --command="VOTRE_SQL_ICI" --remote


      

        
        
Alternative plus simple : Ouvrez dash.cloudflare.com → Workers & Pages → D1 → gestapp-db → Console, et collez le SQL directement.

🔐 Configurer les secrets du Worker

        
JWT_SECRET et SUPERADMIN_EMAIL — stockés chiffrés dans Cloudflare

      

      En attente
      
    

    

      

        Les secrets ne sont jamais visibles dans le code source.
        Ils sont injectés de manière sécurisée dans env.JWT_SECRET et env.SUPERADMIN_EMAIL.
      


      
🎲 Générer JWT_SECRET (clé aléatoire sécurisée)

      

        

          
          
          
        

        

          Clé 256 bits aléatoire — générée localement, jamais envoyée nulle part
        

      


      
📋 Ajouter JWT_SECRET

      

        
# Dans workers/gestapp-api/ :
wrangler secret put JWT_SECRET
# → Collez la clé générée ci-dessus quand le terminal la demande


      
📋 Ajouter SUPERADMIN_EMAIL

      

        
wrangler secret put SUPERADMIN_EMAIL
# → Saisissez l'email : said.hamdaoui1984@gmail.com


      

        
        
Alternative via Dashboard : dash.cloudflare.com → Workers & Pages → gestapp-api → Settings → Variables → Encrypt

🔗 Mettre à jour l'URL du Worker dans les fichiers

        
Remplace l'URL placeholder dans cloudflare-client.js et admin/index.html

      

      En attente
      
    

    

      

        Deux fichiers contiennent le placeholder https://gestapp-api.digital-pro.workers.dev qui doit être remplacé par l'URL réelle de votre Worker.
      


      

        
        
        

          Renseignée automatiquement depuis l'Étape 1
        

      


      

        

          
          gestapp212/js/cloudflare-client.js — ligne 18
        

        

          
const CF_API_BASE = 'https://gestapp-api.digital-pro.workers.dev';

          
const CF_API_BASE = 'https://gestapp-api.XXXX.workers.dev';

        

      


      

        

          
          admin/index.html — ligne ~591
        

        

          
const CF_API = 'https://gestapp-api.digital-pro.workers.dev';

          
const CF_API = 'https://gestapp-api.XXXX.workers.dev';

        

      


      
⚡ Commandes de remplacement automatique

      

        
# Linux / macOS :
sed -i 's|https://gestapp-api.digital-pro.workers.dev|VOTRE_URL_WORKER|g' gestapp212/js/cloudflare-client.js admin/index.html

# Windows PowerShell :
(Get-Content gestapp212/js/cloudflare-client.js) -replace 'gestapp-api.digital-pro.workers.dev','VOTRE_URL_WORKER' | Set-Content gestapp212/js/cloudflare-client.js


      

        
        
Vous pouvez aussi éditer manuellement les deux fichiers et remplacer gestapp-api.digital-pro.workers.dev par votre URL.
